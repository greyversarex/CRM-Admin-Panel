## 1. Кто заказчик и что строим

**Tajik Music Distribution** — таджикский музыкальный дистрибьютор (аналог Symphonic Distribution).
Цель — построить admin-heavy CRM-панель для распределения музыки артистов на DSP (Spotify, Apple Music,
Yandex Music, YouTube Music и т.д.), управления роялти, выплатами, правами и поддержкой.

**Стек:**
- Монорепа на pnpm workspaces.
- Backend: Node.js + Express 5 + TypeScript + Drizzle ORM + PostgreSQL.
- Frontend: React + Vite + TypeScript + TanStack Query + wouter + shadcn/ui (radix + tailwind).
- DB: PostgreSQL (Replit-managed для дева, отдельный VPS для прода).

**Структура монорепы:**
```
artifacts/
  api-server/      — Express бекенд (порт 8080 в деве)
  crm-panel/       — главное React-приложение CRM (порт 5173)
  mockup-sandbox/  — для прототипирования компонентов на Canvas
lib/
  db/              — Drizzle схемы + миграции (pnpm пакет @workspace/db)
```

## 2. Критические правила (НЕ НАРУШАТЬ)

1. **Язык интерфейса — РУССКИЙ.** Все надписи в UI на русском. i18n есть для ru/en, но русский — основной.
2. **БЕЗ ЭМОДЗИ.** Нигде. Ни в коде, ни в UI, ни в коммитах, ни в чате с пользователем.
3. **Прод на Timeweb VPS** (не на Replit). Поэтому:
   - **НИКОГДА не предлагать `suggest_deploy` / Replit Publish.** Пользователь развёртывает сам через docker/pm2 на VPS.
   - Прод-миграции через `pnpm --filter @workspace/db run migrate` (НЕ push).
   - Дев-миграции — через `drizzle-kit push --force`, разрешено.
4. **Ничего не делать молча, если ломается прод-логика.** Сначала спрашивать или предупреждать.
5. **Audit log обязателен** для всех админских мутаций. Обязательно через `auditMutation(req, {...})` — см. раздел про audit ниже.

## 9. Полезные паттерны и конвенции

### Audit log
ВСЕГДА оборачивать админские мутации (POST/PATCH/PUT/DELETE):
```ts
await auditMutation(req, {
  action: "update",          // create | update | delete | reject | approve | ...
  entityType: "release",     // должен быть в ENTITY_ALLOWLIST в audit.ts
  entityId: release.id,      // или null
  before: oldRow ?? null,
  after: newRow ?? null,
});
```
**Каждый новый `entityType` ОБЯЗАТЕЛЬНО добавить в `ENTITY_ALLOWLIST`** в `artifacts/api-server/src/lib/audit.ts`, иначе сохранятся только `id/createdAt/updatedAt` (default-deny).

### Manager permission middleware
```ts
import { requireManagerPermission } from "../lib/manager-permissions";
router.get("/foo", requireRole("admin", "manager"), requireManagerPermission("catalog"), handler);
```
Порядок: `requireAuth` (если нужен) → `requireRole` → `requireManagerPermission` → handler.

### Доступ к API из фронта
Используется `fetch(url, { credentials: "include" })`. Сессия в HTTP-only cookie. Базовый URL — относительный `/api/...` (Vite проксирует в деве, в проде nginx).

### Стиль UI
shadcn/ui компоненты + tailwind. Тёмная тема по умолчанию. Цвета:
- Primary: `hsl(var(--primary))` — фиолетово-синий.
- Sidebar: `hsl(var(--sidebar))` — глубокий тёмный.
- Acent цвета по разделам: emerald (catalog), green (finance), blue (analytics), violet (rights/users), amber (distribution), cyan (crm), rose (comms), yellow (support).

### i18n
Файл: `artifacts/crm-panel/src/lib/i18n.tsx`. ОДИН большой объект `translations.ru` и `translations.en`. Все ключи добавлять в ОБА языка одновременно — иначе будет fallback на key-string.

### TypeScript
- `strict: true` + `noUncheckedIndexedAccess: true` в api-server. Поэтому `req.params["key"]` нужно кастить или проверять `?? ""`.
- Типы из БД импортятся как `import { someTable, type SomeRow } from "@workspace/db"`.
