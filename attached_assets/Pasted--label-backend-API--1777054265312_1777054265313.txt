Нужно переработать интерфейс для роли "label" (лейбл), не трогая backend-логику и API.

Задача: упростить и привести UI к MVP-логике музыкальной дистрибуции.

1. ОБЩЕЕ ПРАВИЛО

* Лейбл — это бизнес-пользователь, не админ
* Он должен видеть ТОЛЬКО свои данные
* Интерфейс должен быть максимально простым и понятным
* Убрать перегруженность и технические разделы

2. ОБНОВИТЬ SIDEBAR (components/sidebar.tsx или аналог)

ОСТАВИТЬ ТОЛЬКО ЭТИ РАЗДЕЛЫ ДЛЯ ROLE = "label":

* Dashboard (/)
* Releases (/releases)
* Artists (/artists)
* Analytics (/analytics)
* Earnings (использовать /royalties и /finance как единый раздел UI)
* Payouts (/payouts)
* Splits (/splits) — если уже реализовано
* Support (/support)
* Profile (/profile)

УБРАТЬ ИЗ SIDEBAR ДЛЯ LABEL:

* Users
* Distribution
* CRM
* Publishing (или скрыть полностью)
* Finance как отдельный технический раздел (оставить только внутри Earnings)
* Videos (если не критично)
* Settings (оставить только profile)

3. DASHBOARD ДЛЯ LABEL

Убедиться, что:

* показываются только данные label scope
* есть:

  * Total Revenue
  * Total Streams

  * Recent Releases

Если есть блоки типа:

* system alerts
* глобальные данные

→ скрыть для label

Оставить только:

9. ПРОВЕРИТЬ BACKEND SCOPE

Убедиться, что:

* все запросы идут с учетом label_id
